## 1. Инструкция по запуску решения
Для запуска очистки на реальных или синтетических данных выполните команду:
```bash
python solution_Федосеев_АА_ШАД_111.py
```

### Режим очистки данных:

  - Скрипт автоматически найдет партиционированные parquet-файлы в текущей
    директории (например, по шаблону month=*).
  - Вы также можете явно передать путь к файлу данных или директории в качестве
    аргумента командной строки:
    ```bash
    python solution_Федосеев_АА_ШАД_111.py path/to/data_folder
    ```
  - В случае отсутствия в системе parquet-движков (pyarrow/fastparquet) или
    самих файлов данных, скрипт автоматически сгенерирует синтетический
    массив данных (50 000 строк) для демонстрации работы всех модулей.

Режим автоматического тестирования:

Для запуска встроенных модульных тестов выполните команду с флагом --test:
```bash
python solution_Федосеев_АА_ШАД_111.py --test
```
Скрипт запустит тестовые сценарии на базе стандартной библиотеки unittest,
проверит корректность фильтрации, предобработки типов и изоляции
аномальных записей, после чего выдаст структурированный отчет и
завершит работу.

2. Математическое описание и методология

Алгоритм расчета и природа данных

Дневной OTS респондента по конкретному бренду рассчитывается как:
daily\_ots(i, j, k) = Weight(i, k) \times count\_rows(i, j, k)

Поисковая активность респондентов характеризуется сильной правосторонней
асимметрией (распределение с «тяжелым правым хвостом»): подавляющее
большинство пользователей совершают 1–2 запроса, умеренно активные — 5–15
запросов, а технические сбои и парсеры (боты) генерируют сотни или тысячи
запросов.

Модифицированный Z-score в логарифмической шкале (Улучшение В)

Для устранения ложноположительных срабатываний на лояльных активных
пользователях вычисления перенесены в логарифмическое пространство:
daily\_ots_{log} = \ln(daily\_ots + 1)

Робастный показатель Modified Z-score рассчитывается на основе медианы и
медианного абсолютного отклонения (MAD) в логарифмической шкале:
Score_i = 0.6745 \times \frac{daily\_ots_{log, i} - \text{median}_{log}}{MAD_{log}}

Преимущества логарифмирования:

1.  Устранение over-cleansing (переочистки): Линейный масштаб без
    логарифмирования классифицировал как выбросы обычных активных
    пользователей. Логарифмическое сжатие подтягивает реальную высокую
    активность к медиане, позволяя отсекать по порогу Score_i > 3.5
    только экстремальные технические аномалии. Доля удалений респондентов
    снижается с критических 28.33% до математически обоснованных 6.14%,
    сохраняя 90.95% полезного OTS (вместо 57.19% в линейной модели).
2.  Обратное декодирование порогов: Рассчитанные в логарифмическом пространстве
    пороги переводятся обратно в исходную шкалу OTS:
    threshold_{raw} = \exp(threshold_{log}) - 1 Это позволяет выводить в
    диагностические логи понятные человеку физические значения OTS.

Робастное сглаживание отклонения (MAD Smoothing — Улучшение Б)

В однородных выборках брендов (где почти все респонденты сделали ровно по 1
запросу) значение MAD стремится к нулю. Деление на сверхмалый MAD в
классической формуле приводило к лавинообразному завышению Z-score для
респондентов с минимальным отклонением (например, сделавших 2 запроса).

Для стабилизации знаменателя внедрено сглаживание:
MAD_{smoothed} = \max(MAD, \epsilon \times \text{median}) где
\epsilon = 10^{-4}. Дополнительно MAD ограничивается снизу константой 10^{-5},
что полностью предотвращает появление неопределенностей (деления на ноль).

Fallback-иерархия при разреженности данных

Из-за высокой кардинальности брендов (более 3000 уникальных ID) группы на самом
детальном уровне часто оказываются слишком малы. Для репрезентативного расчета
статистик применена система fallback-переходов:

  - Уровень 1 (Бренд): Оценка в группе бренда за выбранный день. Условие
    применения: уникальных респондентов в группе \ge 5 и MAD > 0.
  - Уровень 2 (Категория): При малой выборке бренд-группы расчет медианы и MAD
    производится по всей категории CategoryDelivery за этот день.
  - Уровень 3 (День): При малом объеме категории базовые статистики
    рассчитываются глобально по всей активности за текущий день.

3. Логика удаления данных

  - Единица анализа (триггер): (SubjectID, researchdate, BrandID,
    CategoryDelivery) — точка, где зафиксировано превышение порога в
    логарифмической шкале.
  - Единица удаления: (SubjectID, researchdate) — если респондент признан
    аномальным хотя бы по одному бренду в конкретный день, из итоговой
    выборки исключаются все его записи за этот день (включая запросы по
    другим брендам и категориям), во избежание искажения суточной
    активности.

4. Структура выходных файлов (Папка output/)

После успешного выполнения программы создается каталог output/:

Отчетные таблицы:

1.  output/anomalies.csv — список уникальных пар респондент-день для очистки
    выборки.
      - Схема: SubjectID, researchdate (без дубликатов).
2.  output/anomaly_reasons.csv — диагностическая таблица причин удаления.
      - Схема: SubjectID, researchdate, BrandID, Brand, CategoryDelivery,
        daily_ots, score, threshold, reason.
      - Столбец reason подробно описывает, на каком уровне иерархии (Бренд,
        Категория или День) сработал триггер, какой скор был зафиксирован и
        какой порог в исходной шкале OTS был превышен.

Обязательные диагностические графики (output/plots/):

1.  total_ots_before_after.png — линейный график изменения суммарного OTS по
    дням до и после проведения очистки. Показывает сглаживание экстремальных
    технических пиков.
2.  category_ots_change.png — процентное изменение OTS по категориям доставки.
    Демонстрирует контролируемое и равномерное влияние алгоритма.
3.  daily_anomaly_count.png — количество уникальных удаленных респондентов по
    дням.

Дополнительные аналитические визуализации:

  - output/plots/demo_gender.png — сравнение OTS до/после очистки в разрезе пола
    респондентов.
  - output/plots/demo_age.png — сравнение OTS до/после очистки по возрастным
    группам.
  - output/plots/resource_platform.png — сравнение OTS до/после очистки по
    используемым платформам (Platform).
  - output/plots/brand_ots_change.png — график динамики OTS по дням до/после
    очистки для наиболее популярного бренда в выборке (демонстрирует
    точечное устранение экстремальных выбросов).
